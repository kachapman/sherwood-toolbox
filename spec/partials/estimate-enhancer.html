Implementation: Estimate Enhancer

Port the existing Flask app into a mountable blueprint.

What it does

Takes a construction estimate PDF and returns an enhanced copy: spell-checked against a construction vocabulary, zero-quantity line items flagged, clickable links between "see image" references and the photo pages, user terms highlighted, and selected IRC code-reference PDFs appended.

Source to port

  • EstimateEnhancer/pdf-app/utils/ (markup_bridge, spell_utils, spell_vocab)
  • EstimateEnhancer/pdf-app/templates/index.html, static/
  • EstimateEnhancer/pdf-app/Add Image Links Fork (subprocess helper)

Coupling fixes (the only real work)

  1. Absolute URLs in template JS. templates/index.html hardcodes /analyze, /process, /download/, /uploads/ (lines 616, 771, 809, 815) and /static/ (lines 7, 364). Inject the blueprint base path into the page (a data-base attribute or a JS const from url_for) and build requests as ${BASE}/analyze. Swap static refs to url_for('estimate_enhancer.static', ...).
  2. Image-link payload. build_image_link_payload emits an absolute /uploads/{file}#page=N (app.py:765). Prefix it with the blueprint base.
  3. Filesystem paths. UPLOAD_FOLDER='uploads' and ATTACHMENTS_FOLDER='attachments' (app.py:57-58) are cwd-relative. Read them from toolbox/config.py (absolute paths).
  4. Fork path. process_with_fork resolves the helper relative to the file then falls back to ~/Documents/App Projects/... (app.py:1260-1262). Replace with a single ADD_IMAGE_LINKS_FORK_PATH from config pointing at tools/estimate_enhancer/fork/. It is invoked python3 <fork> <input_pdf> with PDF_HIGHLIGHT_RULES_JSON in the env.

Dependencies

PyMuPDF (fitz), pypdf, pyspellchecker. All have Linux wheels.

Web server deployment (parked)

Benched, not deleted. Preserved here so it can be revived without re-deriving it.

Shape

The same modular-monolith Flask app, deployed as a single container instead of a local launcher. One image runs the app under a production WSGI server (gunicorn); a reverse proxy sits in front only for TLS and the domain.

deploy/
  Dockerfile          single image: gunicorn toolbox.wsgi:app
  docker-compose.yml  app + reverse proxy (TLS/domain only)
  proxy/              nginx or Traefik config

Key points to remember when reviving

  • Run gunicorn toolbox.wsgi:app (the same create_app(), with OFFLINE unset so network features are live).
  • One volume for ~/.config/restoration_toolkit (companies, signatures) and one for Estimate Enhancer uploads/attachments.
  • The proxy terminates TLS and serves the domain; the app itself stays HTTP on an internal port.
  • Because the app is one process serving all tools at path prefixes (/estimate-enhancer, /iws, /photo-report, /documents), no per-tool routing config is needed; the earlier multi-container proxy-hub is unnecessary.

Why parked

The local version covers the immediate need and works offline. The web deployment is recorded for when multi-user or remote access is wanted.

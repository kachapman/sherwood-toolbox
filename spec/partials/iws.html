Implementation: Ice and Water Shield Calculator

Pure static. The least work of the four.

What it does

Client-side calculator: from roof size, pitch, eave and valley lengths, soffit depth, wall and stud parameters it computes ice-and-water-shield coverage in two modes (Actual SF and Full Roll). All math runs in the browser; no backend.

Source to vendor

  • ~/.local/src/SyncedViruses/VanguardIWSCalc/src/ -> tools/iws/static/ (index.html, css/, js/calculator.js, assets/).

The src/ already carries the Sherwood logo and the title "Ice Barrier Calculator"; it contains zero vanguard strings. The rename is satisfied by not carrying the old VanguardIWSCalc directory name into the toolbox.

Target

tools/iws/
  __init__.py     blueprint, serves the calculator under /iws
  static/         the vendored src (html/js/css/assets)

The blueprint serves static/index.html at /iws and the assets beside it. To share the toolbox chrome, wrap the calculator body in base.html; to keep it verbatim, serve it as-is inside the content frame. Wrapping is preferred for a consistent sidebar.

Input fields (mirror in the chrome if wrapped)

Project name, address, roof size (SQ), inside exterior wall (in), predominant pitch (/12), eave length (LF), valley length (LF), soffit depth (in), stud size (2x4 / 2x6), exterior type (vinyl / wood / stucco / brick), calculation mode (eave+valley / eave only).

Edit point

The coverage math lives in tools/iws/static/js/calculator.js. Change the formula there; no Python involved.

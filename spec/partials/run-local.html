Run locally

Install (one time, needs network)

cd ~/Documents/projects/SherwoodToolbox
./run/install-standalone.sh

The script creates a venv, runs pip install . (toolbox + restoration-common; PyQt5 is not pulled in because only the headless core is imported), installs sherwood-toolbox.desktop into ~/.local/share/applications/, appends a sherwood-toolbox alias to ~/.zshrc, and records the alias in opencodeAssets/aliasDirectory.txt.

Launch (offline after install)

  • GNOME: open "Sherwood Toolbox" from the app grid.
  • Terminal: sherwood-toolbox.

The launcher runs run/standalone.py: create_app(offline=True), waitress on a free 127.0.0.1 port, then opens the default browser. Serving over http://127.0.0.1 avoids the Flatpak file:// sandbox that blocks sibling files, so multi-page navigation and uploads work normally.

Offline behavior

core/capabilities.py checks network reachability, CRM credentials (~/.config/photo_report_generator/crm.ini), and the fork script. Templates read those flags and disable or annotate features rather than erroring. "Fetch from CRM" returns a clear "offline or not configured" message; every other feature works with no network. The launcher sets OFFLINE=1 to skip network probes entirely.

Data locations

  • Companies and signatures: ~/.config/restoration_toolkit/.
  • CRM credentials: ~/.config/photo_report_generator/crm.ini.
  • Shipped default companies and logos: inside the restoration_common package.

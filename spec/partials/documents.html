Implementation: Documents

New blueprint; two PDFs per submit. No PyQt5.

What it does

Generates an Invoice PDF and a Certificate of Completion PDF from job info, line items, and an optional signature. Line items can include a base charge, preset ITEL (199.80) and NTS (150.00) lines, custom rows, or a CSV import, with a running grand total.

Existing code to call (headless, no Qt)

  • restoration_common.InvoicePDFGenerator(output_path, company_config, job_info, line_items, logo_path=None, signature_path=None, signature_name=None) then .generate().
  • restoration_common.COCPDFGenerator(...) then .generate() (same inputs).
  • load_companies, get_company_by_id, get_signature_path / save_signature_for_company, fetch_job_info_from_url.

job_info dict keys

customer_name, street, city_state_zip, job_number (required), insurance_claim, sales_rep, note, invoice_number (auto from job_number), invoice_date (default today), terms (default from company), invoice_notes, base_charge ({description, amount}).

line_items

A list of (description, amount) tuples assembled from: the base charge (if non-zero), the ITEL and NTS checkboxes, custom table rows, and any CSV import.

Target

tools/documents/
  __init__.py     blueprint, url_prefix=/documents
  routes.py       POST -> Invoice + COC generate -> send_file(zip)
  forms.py        fields, line-item table, signature handling
  templates/      form (layout seed: mockup documents page)

Signature

Saved signatures are base64 PNGs in ~/.config/restoration_toolkit/signatures.json; resolve with get_signature_path to a temp PNG and pass as signature_path. A custom upload overrides it.

Output

Generate both PDFs to a temp dir, zip them, send_file the zip. Validation: customer_name and job_number required.

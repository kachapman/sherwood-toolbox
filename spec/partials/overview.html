Overview and architecture

What this file is

A self-contained build spec for the Sherwood Toolbox. It bundles the signed-off visual mockup with the instructions to make each tool work locally, plus a parked appendix for the benched web-server deployment. It opens in any browser with no server and no external requests.

The toolbox

One Flask app presents four tools behind a shared hub: Estimate Enhancer, Ice and Water Shield Calculator, Photo Report, and Documents. Each tool is an isolated blueprint package under toolbox/tools/<name>/. The app runs locally: a launcher serves it on 127.0.0.1 and opens the browser.

The registry is the keystone

toolbox/registry.py is the single source of truth: a list of tool specs (id, label, icon, url_prefix, blueprint, offline_capable, description). The hub tiles, the sidebar nav, and blueprint registration all read it. Adding a tool means dropping a package and adding one registry entry.

Reused, not rebuilt

  • Estimate Enhancer: the existing Flask routes and PDF logic, refactored into a blueprint.
  • IWS: the existing static calculator, vendored in.
  • Photo Report and Documents: new web forms over the existing headless restoration_common generators (zero PyQt5).

Run modes

Active: local standalone (this spec). Parked: web server (see the Web Server appendix), preserved so it can be revived without re-deriving it.
